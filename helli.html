1 Project Overview

TravelNest is a fully client-side Tourist Guide Web Application built without any backend or database. It allows users to discover, explore, save, and plan trips to popular tourist destinations across India — entirely within the browser.

All user data (favourites, trip plans, dark mode preference, recently viewed destinations) is stored persistently using the browser's localStorage API, meaning data survives page refreshes and browser restarts without requiring a server, login, or cloud storage.

What Problem Does It Solve?

Problem TravelNest Solution
Scattered travel information Centralised destination cards with full details, tips, gallery, and maps
No offline trip planning tool LocalStorage-powered trip planner — works completely offline
Generic travel websites Personalised recommendations + recently viewed history
Complex trip management Simple planner with add / edit / delete and countdown badges
Poor mobile travel browsing Fully responsive layout across mobile, tablet, and desktop

2 Key Benefits

For Users

  • Zero sign-up required — No account, no login, no password. Open and use instantly.
  • Works offline — Once loaded, the app works without an internet connection (except map embeds and images).
  • Data never disappears — Saved destinations and trip plans persist in LocalStorage permanently until manually cleared.
  • Fast and lightweight — No backend calls, no API latency. All filtering and searching is instant, client-side.
  • Privacy-first — No user data is sent to any server. Everything stays on the user's own device.
  • Dark mode — Reduces eye strain during night browsing; preference is saved automatically.
  • Native share — Uses the Web Share API on mobile, or clipboard copy on desktop.

For Developers & Students

  • No installation required — Pure HTML, CSS, and JavaScript. Just open the file in any browser.
  • Clean, modular code — Each feature (storage, rendering, filtering, modal) is a clearly separated function.
  • Easy to extend — Add new destinations by appending to the DESTINATIONS JSON array. No database migration needed.
  • Great learning project — Demonstrates SPA routing, LocalStorage CRUD, DOM manipulation, event handling, and responsive design without any framework.

5 Features & How They Work

Single Page Application (SPA) Routing

TravelNest simulates multi-page navigation without URL changes or page reloads. All 5 pages are <div> elements in the DOM, toggled via JavaScript. 

function showPage(p) {
  // Hide all pages
  PAGES.forEach(id => {
    document.getElementById('page-' + id).classList.toggle('hidden', id !== p);
  });
  window.scrollTo({ top: 0, behavior: 'smooth' });
  if (p === 'home')     renderHome();
  if (p === 'explore')  renderExplore();
  if (p === 'favorites') renderFavorites();
  if (p === 'planner')  renderPlans();
}

Real-Time Search & Filter System

The Explore page features a multi-criteria filter system with instant results.

  • User types or changes a dropdown → triggers renderExplore()
  • Debounced by 350ms using setTimeout to prevent lag on fast typing
  • All filters applied simultaneously using Array.filter()
  • Results sorted using Array.sort() by rating or name
  • DOM updated by injecting HTML strings into #exploreGrid

Image Slider

Each destination modal contains a smooth 3-image gallery slider.

  • Images laid out in a flex row inside .slider-track
  • transform: translateX(-N * 100%) shifts the visible image
  • window._sliderIdx tracks the current index globally
  • Dot indicators update by toggling the .active class

Toast Notification System

Non-intrusive feedback messages appear bottom-right for every user action.

  • toast(msg, type) creates a <div> and appends it to #toast-container
  • CSS @keyframes handles slide-in (toastIn) and fade-out (toastOut) animations
  • setTimeout removes the element from the DOM after 3.2 seconds
  • Three types: success  error  info

Recently Viewed & Recommendations

  • Recently Viewed: Every openDetail(id) call prepends the ID to tn_recent in LocalStorage. Duplicates are removed, capped at 8 items.
  • Recommendations: Destinations outside the top-6 featured list are randomly shuffled using .sort(() => Math.random() - 0.5). The first 4 are shown.

6 Data Architecture

All destination data is stored as a JavaScript array of objects in the source file. No external API or database is required.

Destination Object Schema

idNumberUnique numeric identifier
nameStringDestination name (e.g. "Taj Mahal")
locationStringCity and state (e.g. "Agra, Uttar Pradesh")
categoryString"beach" | "hill" | "historical" | "temple" | "nature" | "city"
ratingNumberDecimal rating from 0.0 to 5.0 (e.g. 4.9)
descriptionStringFull destination description paragraph
imageStringPrimary card image URL
galleryString[]Array of 3 image URLs for the modal slider
best_timeStringBest months to visit (e.g. "October – March")
nearbyString[]List of nearby attractions
tipsStringTravel tips and local advice for visitors
map_linkStringGoogle Maps embed URL for the map tab
➕
Adding New DestinationsSimply append a new object to the DESTINATIONS array following the schema above. No other code changes are required — the new destination will automatically appear in all grids, filters, and search results.

7 LocalStorage System

Storage Keys

Key Type Contents
tn_favs Array<Number> Array of saved destination IDs
tn_plans Array<Object> Array of trip plan objects
tn_dark Boolean Dark mode preference (true/false)
tn_recent Array<Number> Recently viewed destination IDs (max 8)

Trip Plan Object Schema

idNumberTimestamp used as unique ID: Date.now()
destStringDestination name (free text input)
dateStringISO date string: "YYYY-MM-DD"
notesStringOptional notes / reminders
budgetStringOptional budget amount in ₹
modeStringTravel mode emoji string (e.g. "✈️ Flight")

Storage Helper API

const S = {
  get:      (key) => JSON.parse(localStorage.getItem(key)),
  set:      (key, v) => localStorage.setItem(key, JSON.stringify(v)),
  getFavs:  () => S.get('tn_favs')   || [],
  setFavs:  (v) => S.set('tn_favs', v),
  getPlans: () => S.get('tn_plans')  || [],
  setPlans: (v) => S.set('tn_plans', v),
  getDark:  () => S.get('tn_dark')   || false,
  setDark:  (v) => S.set('tn_dark', v),
  getRecent:() => S.get('tn_recent') || [],
  setRecent:(v) => S.set('tn_recent', v),
};

All reads return || [] or || false as a fallback, so the app never crashes on a fresh browser with no existing data.

9 Page-by-Page Breakdown

Home Page

  • Hero Section — Full-viewport background image, gradient overlay, animated eyebrow label, fluid headline, search box with Enter key support, and quick-category pill buttons
  • Stats Bar — 4 key numbers: 50+ destinations, 4.9★ rating, 12K+ users, 6 categories
  • Featured Destinations — Top 6 by rating, rendered as full destination cards
  • Recommended — 4 randomly selected from remaining destinations
  • Recently Viewed — Pulled from LocalStorage; empty state shown when no history

Explore Page

  • Multi-criteria filter bar (text search + category + min rating + sort order)
  • Category tab row for quick one-click filtering
  • Debounced real-time rendering (350ms delay prevents excessive DOM updates)
  • Empty state displayed when no results match current filters

Saved (Favorites) Page

  • Reads tn_favs from LocalStorage and maps IDs to destination objects
  • Cards include a remove button to delete individual saved items
  • Empty state with a CTA button that navigates to the Explore page

Trip Planner Page

  • Form fields: destination, travel date, budget (₹), travel mode, notes
  • Dual-mode form: Add New Trip / Edit Trip controlled by editingPlanId
  • Plans sorted by travel date (ascending, nearest first)
  • Countdown badge: Xd away Today! Past
  • Edit populates the form and smoothly scrolls to the top

About Page

  • Stats grid with brand-coloured icon tiles
  • Three feature cards: Discover, Save & Plan, Share

Destination Detail Modal

  • Opens on card "Explore" button click from any page
  • 3-image gallery slider with dot indicators and arrow buttons
  • Three tabs: Overview (description, best time, nearby), Travel Tips, Map (Google Maps embed)
  • Action buttons: Plan This Trip (opens quick planner modal), Share
  • Bookmark toggle synced across all visible cards in real time

11 How to Run the Project

Method 1 — Direct File Open (Simplest)

1. Download or copy the index.html file to your computer
2. Double-click the file to open it in any modern browser
3. Done — no installation, no server, no terminal required

Method 2 — VS Code Live Server

1. Open the project folder in Visual Studio Code
2. Install the "Live Server" extension by Ritwick Dey
3. Right-click index.html → "Open with Live Server"
4. App opens at http://127.0.0.1:5500

Method 3 — GitHub Pages

1. Create a new GitHub repository
2. Upload index.html to the root of the repository
3. Go to Settings → Pages → Source: main branch / root
4. App is live at https://yourusername.github.io/repository-name

Method 4 — Netlify (Drag & Drop)

1. Go to netlify.com → "Add new site" → "Deploy manually"
2. Drag and drop your project folder into the upload zone
3. A live public URL is generated instantly — no configuration needed
⚠️
Internet Required ForTailwind CSS CDN, Font Awesome CDN, destination images (Unsplash), and Google Maps embeds require an internet connection. All app logic, routing, filtering, and LocalStorage features work fully offline once initial assets are cached.
